Element  <security> - Zugriffsschutz

<security> gibt die Rechner und Netzwerke an, die per TCP und UDP Kommandos ausführen dürfen.

schicken können Sie eine Basiskonfiguration angegeben, die der Scheduler vor der Konfiguration verarbeitet, die <base> enthält. Erst danach verarbeitet er die Konfiguration, die <base> enthält. In der Basiskonfiguration können Sie Einstellungen vorgenommen, die die übergeordnete Konfiguration ergänzt.

Eine Basiskonfiguration kann auf eine weitere Basiskonfiguration verweisen. Der Scheduler prüft nicht, ob eine Basiskonfiguration selbstbezüglich aufgerufen wird.

Eltern-Elemente

<config>

Attribute

ignore_unknown_hosts="yes|no"

Bei ignore_unknown_hosts="no" führen nicht auflösbare Hostnamen (Element <host>) zum Abbruch des Schedulers. Der Scheduler startet nicht. Das ist derzeit die Voreinstellung.

ignore_unknown_hosts="yes" verhindert den Abbruch, der Scheduler ignoriert nicht auflösbare Hostnamen.

Kind-Elemente

<host> hat keine Elemente.
